# Serveurs MCP — Agregateurs

Serveurs MCP qui connectent a de multiples services via une seule interface. Au lieu d'installer un serveur MCP par service, un agregateur fournit un point d'entree unique vers des centaines ou milliers d'APIs.

---

| Serveur | Description | Nombre d'APIs | Stars |
|---------|-------------|--------------|-------|
| [Pipedream](https://github.com/PipedreamHQ/pipedream) | Connecter 2 500 APIs avec 8 000+ outils preconstruits | 2 500 | ~11k |
| [Composio / Rube](https://composio.dev/) | Connecter a 500+ apps | 500+ | — |
| [Knit MCP](https://www.getknit.dev/) | 10 000+ integrations (HRIS, ATS, CRM) | 10 000+ | — |
| MetaMCP | Middleware MCP unifie avec interface de gestion GUI | Variable | ~2.1k |
| [1mcp/agent](https://github.com/1mcp/agent) | Unifier plusieurs serveurs MCP en un seul | Variable | — |

---

## Quand utiliser un agregateur

- Vous avez besoin de connecter Claude a **plusieurs services** (Gmail + Slack + GitHub + Notion)
- Vous ne voulez pas gerer **la configuration de multiples serveurs MCP** individuellement
- Vous avez besoin d'un **point d'entree unique** pour des actions cross-services
