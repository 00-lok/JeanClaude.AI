## Ce que ca fait

Le Compound Engineering part du principe que chaque unite de travail doit rendre les suivantes plus faciles. 6 commandes couvrent le cycle complet : ideation → brainstorm → plan → execution → revue → capitalisation.

Le point fort est `/ce:review` : **12 subagents paralleles** examinent le code chacun sous un angle different (securite, performance, sur-ingenierie, accessibilite, design API, etc.). La revue est la plus profonde de l'ecosysteme.

`/ce:compound` capture les apprentissages en documentation pour les sessions futures — l'effet compose.

---

## Fonctionnalites cles

- **`/ce:ideate`** — Ideation divergente avec filtrage adversarial
- **`/ce:brainstorm`** — Explorer exigences et approches
- **`/ce:plan`** — Convertir les idees en plans d'implementation detailles
- **`/ce:work`** — Executer les plans avec worktrees et suivi de taches
- **`/ce:review`** — 12 subagents paralleles, chacun sur une perspective differente
- **`/ce:compound`** — Capturer les apprentissages en docs pour les sessions futures
